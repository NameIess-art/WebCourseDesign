# MallSystem 前后端分离核心业务代码流程说明

本文档说明 MallSystem 在前后端分离架构中，登录注册、添加商品、购买商品、订单生成四条核心链路的完整代码流程。重点包括：前端如何组装参数、Axios 如何传输数据、后端如何接收和校验参数、Service 如何执行业务逻辑、订单如何生成，以及相关代码位置。

## 1. 总体架构与数据传输方式

项目采用 Vue 3 + Axios 前端、Spring Boot 3 + Spring Security + JWT + Spring Data JPA 后端的前后端分离架构。

### 1.1 前端请求统一入口

前端所有业务请求都通过 Axios 实例发送：

- `frontend/src/api/http.js`

核心配置：

```js
const http = axios.create({
  baseURL: '/api',
  timeout: 10000
})
```

前端页面不直接写完整后端地址，而是调用 `mall.js`、`admin.js` 中封装好的函数。开发环境下 Vite 会把 `/api` 代理到后端服务：

- `frontend/vite.config.js`
- 目标地址：`http://localhost:18080`

### 1.2 Token 和商家上下文传输

登录成功后，前端将后端返回的 JWT 和用户信息保存到 `localStorage`：

- `frontend/src/utils/auth.js`
- `saveAuth(payload)`
- `getToken()`
- `getUser()`

之后每次请求都会在 Axios 请求拦截器中自动附加认证头：

```js
Authorization: Bearer <token>
```

代码位置：

- `frontend/src/api/http.js`
- `http.interceptors.request.use(...)`

商家/管理员后台还会附加商家上下文：

```js
X-Merchant-Id: <merchantId>
```

后端根据 JWT 识别当前用户，根据 `X-Merchant-Id` 判断当前正在操作哪个商家。

### 1.3 后端统一响应格式

后端所有接口返回 `ApiResponse<T>`：

- `backend/src/main/java/com/mall/vo/ApiResponse.java`

成功响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

失败响应：

```json
{
  "code": 400,
  "message": "错误原因",
  "data": null
}
```

统一异常处理位置：

- `backend/src/main/java/com/mall/exception/GlobalExceptionHandler.java`

其中：

- `BusinessException` 会转成 HTTP 400。
- `MethodArgumentNotValidException` 会把 Bean Validation 参数校验错误转成 HTTP 400。
- 其他未知异常转成 HTTP 500。

## 2. 登录流程

### 2.1 前端页面入口

代码位置：

- `frontend/src/views/LoginView.vue`

用户在登录表单中输入：

- `username`
- `password`

点击登录按钮后触发：

```js
async function submit()
```

前端调用：

```js
const res = await login(form)
```

`login` 的封装位置：

- `frontend/src/api/mall.js`

```js
export const login = (payload) => http.post('/auth/login', payload)
```

最终请求：

```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "demo",
  "password": "123"
}
```

### 2.2 后端 Controller 接收参数

代码位置：

- `backend/src/main/java/com/mall/controller/AuthController.java`

入口方法：

```java
@PostMapping("/login")
public ApiResponse<AuthResponse> login(@Valid @RequestBody LoginRequest request)
```

关键点：

- `@RequestBody`：把前端 JSON 反序列化为 `LoginRequest`。
- `@Valid`：触发参数校验。
- 返回 `AuthResponse`，其中包含 JWT 和用户信息。

### 2.3 登录参数校验

代码位置：

- `backend/src/main/java/com/mall/dto/LoginRequest.java`

```java
public record LoginRequest(
        @NotBlank String username,
        @NotBlank String password
) {
}
```

校验规则：

- `username` 不能为空。
- `password` 不能为空。

如果参数为空，Spring 会抛出 `MethodArgumentNotValidException`，再由 `GlobalExceptionHandler` 转成统一错误响应。

### 2.4 登录业务逻辑

代码位置：

- `backend/src/main/java/com/mall/service/AuthService.java`
- 方法：`login(LoginRequest request)`

执行步骤：

1. 根据用户名查询用户。
2. 使用 BCrypt 比对前端传入的明文密码和数据库中的加密密码。
3. 密码正确后生成 JWT。
4. 返回 `AuthResponse`。

关键代码逻辑：

```java
UserAccount user = userMapper.findByUsername(request.username())
        .orElseThrow(() -> new BusinessException("Invalid username or password"));

if (!passwordEncoder.matches(request.password(), user.getPassword())) {
    throw new BusinessException("Invalid username or password");
}

String token = jwtService.generateToken(user.getId(), user.getUsername(), user.getRole().name());
```

JWT 生成位置：

- `backend/src/main/java/com/mall/security/JwtService.java`
- 方法：`generateToken(...)`

JWT 内容包含：

- `subject`：用户名
- `userId`
- `role`
- 签发时间
- 过期时间

### 2.5 前端保存登录态并跳转

代码位置：

- `frontend/src/views/LoginView.vue`
- `frontend/src/utils/auth.js`

前端收到响应后：

```js
saveAuth(res.data)
```

保存：

- `mall_token`
- `mall_user`

然后根据用户角色跳转：

- `ADMIN` -> `/platform`
- `MERCHANT` -> `/merchant`
- 普通用户 -> `/`

路由守卫位置：

- `frontend/src/router/index.js`

受保护页面会检查本地是否有 token，没有 token 则跳转到 `/login`。

## 3. 注册流程

### 3.1 前端页面入口

代码位置：

- `frontend/src/views/RegisterView.vue`

表单字段：

- `username`
- `email`
- `password`
- `confirmPassword`

点击注册时触发：

```js
async function submit()
```

前端先做一次基础校验：

```js
if (form.password !== form.confirmPassword) {
  window.alert('两次输入的密码不一致')
  return
}
```

然后组装注册 payload：

```js
const payload = {
  username: form.username,
  password: form.password,
  displayName: form.username,
  email: form.email
}
```

调用：

```js
await register(payload)
```

接口封装位置：

- `frontend/src/api/mall.js`

```js
export const register = (payload) => http.post('/auth/register', payload)
```

最终请求：

```http
POST /api/auth/register
Content-Type: application/json

{
  "username": "new_user",
  "password": "123456",
  "email": "new_user@example.com",
  "displayName": "new_user"
}
```

### 3.2 后端 Controller 接收注册请求

代码位置：

- `backend/src/main/java/com/mall/controller/AuthController.java`

入口：

```java
@PostMapping("/register")
public ApiResponse<Void> register(@Valid @RequestBody RegisterRequest request)
```

### 3.3 注册参数校验

代码位置：

- `backend/src/main/java/com/mall/dto/RegisterRequest.java`

校验规则：

```java
@NotBlank @Size(min = 3, max = 50) String username
@NotBlank @Size(min = 6, max = 100) String password
@NotBlank @Email String email
@NotBlank @Size(min = 2, max = 100) String displayName
```

校验含义：

- 用户名不能为空，长度 3 到 50。
- 密码不能为空，长度 6 到 100。
- 邮箱不能为空，并且必须符合邮箱格式。
- 展示名不能为空，长度 2 到 100。

### 3.4 注册业务逻辑

代码位置：

- `backend/src/main/java/com/mall/service/AuthService.java`
- 方法：`register(RegisterRequest request)`

执行步骤：

1. 检查用户名是否已存在。
2. 检查邮箱是否已存在。
3. 使用 BCrypt 加密密码。
4. 创建 `UserAccount`。
5. 默认角色设置为 `USER`。
6. 保存到数据库。

对应实体：

- `backend/src/main/java/com/mall/entity/UserAccount.java`

## 4. 添加商品流程

添加商品属于商家/管理员后台功能，不是普通用户前台功能。

### 4.1 前端页面入口

代码位置：

- `frontend/src/views/MerchantView.vue`

商品表单绑定对象：

```js
const productForm = reactive(defaultProduct())
```

表单包含：

- 商品名称：`name`
- 副标题：`subtitle`
- 描述：`description`
- 图片地址：`imageUrl`
- 价格：`price`
- 总库存：`stock`
- 分类 ID：`categoryId`
- 上下架状态：`active`
- 促销标签：`promotionTag`
- SKU 列表：`skus`
- 图文详情：`detailBlocks`

保存商品触发：

```js
async function saveProduct()
```

如果是新商品：

```js
await createProduct(productForm)
```

如果是编辑商品：

```js
await updateProduct(editingProductId.value, productForm)
```

接口封装位置：

```js
export const createProduct = (payload) => http.post('/admin/products', payload)
```

### 4.2 添加商品权限控制

代码位置：

- `backend/src/main/java/com/mall/config/SecurityConfig.java`

```java
.requestMatchers("/api/admin/**").hasAnyRole("ADMIN", "MERCHANT")
```

表示 `/api/admin/**` 下的接口必须是管理员或商家角色才能访问。

### 4.3 后端 Controller 接收商品请求

代码位置：

- `backend/src/main/java/com/mall/controller/AdminController.java`

入口：

```java
@PostMapping("/products")
public ApiResponse<ProductResponse> createProduct(@Valid @RequestBody ProductRequest request)
```

关键点：

- `@RequestBody` 接收商品 JSON。
- `@Valid` 执行参数校验。
- 调用 `adminService.createProduct(request)`。

### 4.4 商品参数校验

代码位置：

- `backend/src/main/java/com/mall/dto/ProductRequest.java`

主要校验规则：

```java
@NotBlank String name
@NotBlank String subtitle
@NotBlank String description
@NotBlank String imageUrl
@NotNull @DecimalMin("0.01") BigDecimal price
@NotNull @Min(0) Integer stock
@NotNull Long categoryId
@NotNull Boolean active
```

含义：

- 商品基础文本不能为空。
- 商品价格必须大于等于 0.01。
- 库存不能小于 0。
- 分类 ID 不能为空。
- 上下架状态不能为空。

SKU 子项校验位置：

- `backend/src/main/java/com/mall/dto/ProductSkuRequest.java`

详情块校验位置：

- `backend/src/main/java/com/mall/dto/ProductDetailBlockRequest.java`

### 4.5 添加商品业务逻辑

代码位置：

- `backend/src/main/java/com/mall/service/AdminService.java`
- 方法：`createProduct(ProductRequest request)`

执行步骤：

1. 读取当前商家 ID。
2. 根据 `categoryId` 查询分类，分类不存在则抛出业务异常。
3. 创建 `Product` 实体。
4. 设置商家、分类、价格、库存、上下架、促销标签等字段。
5. 保存商品主表。
6. 同步保存 SKU 列表。
7. 同步保存图文详情块。
8. 返回 `ProductResponse` 给前端。

关键方法：

- `createProduct(...)`
- `updateFields(...)`
- `syncProductChildren(...)`
- `defaultSku(...)`
- `defaultDetailBlock(...)`

对应实体：

- `backend/src/main/java/com/mall/entity/Product.java`
- `backend/src/main/java/com/mall/entity/ProductSku.java`
- `backend/src/main/java/com/mall/entity/ProductDetailBlock.java`

如果前端没有传 SKU，后端会根据商品主表信息自动生成默认 SKU；如果没有传详情块，后端会自动生成默认文本详情块。

## 5. 购买商品与加入购物车流程

### 5.1 商品详情页加载商品

代码位置：

- `frontend/src/views/ProductView.vue`

页面加载时执行：

```js
onMounted(loadProduct)
```

`loadProduct` 同时请求：

- 商品详情：`getProduct(route.params.id)`
- 商品评价：`getProductReviews(route.params.id)`
- 商品问答：`getProductQuestions(route.params.id)`

商品详情接口封装：

- `frontend/src/api/mall.js`

```js
export const getProduct = (id) => http.get(`/products/${id}`)
```

后端入口：

- `backend/src/main/java/com/mall/controller/MallController.java`

```java
@GetMapping("/products/{id}")
public ApiResponse<ProductResponse> product(@PathVariable Long id)
```

后端服务：

- `backend/src/main/java/com/mall/service/MallService.java`
- 方法：`product(Long id)`

该方法会校验商品是否存在、是否处于上架状态，然后返回商品、SKU 和详情块。

### 5.2 前端加入购物车

代码位置：

- `frontend/src/views/ProductView.vue`
- 方法：`add()`

前端校验：

1. 未登录则提示先登录。
2. 数量必须大于等于 1。
3. 数量不能超过当前 SKU 库存。

组装请求：

```js
await addCart({
  productId: product.value.id,
  skuId: selectedSku.value?.id ?? null,
  quantity: quantity.value
})
```

接口封装：

- `frontend/src/api/mall.js`

```js
export const addCart = (payload) => http.post('/cart', payload)
```

最终请求：

```http
POST /api/cart
Authorization: Bearer <token>
Content-Type: application/json

{
  "productId": 3,
  "skuId": 5,
  "quantity": 1
}
```

### 5.3 后端加入购物车入口

代码位置：

- `backend/src/main/java/com/mall/controller/MallController.java`

```java
@PostMapping("/cart")
public ApiResponse<Void> addToCart(@AuthenticationPrincipal UserAccount user,
                                   @Valid @RequestBody CartItemRequest request)
```

参数来源：

- `@AuthenticationPrincipal UserAccount user`：由 JWT Filter 解析 token 后放入 Spring Security 上下文。
- `@RequestBody CartItemRequest request`：前端提交的商品和数量。
- `@Valid`：校验 `CartItemRequest`。

### 5.4 加入购物车参数校验

代码位置：

- `backend/src/main/java/com/mall/dto/CartItemRequest.java`

```java
@NotNull Long productId
Long skuId
@NotNull @Min(1) Integer quantity
```

含义：

- 商品 ID 必须传。
- SKU 可以为空，后端会取默认 SKU。
- 数量必须至少为 1。

### 5.5 加入购物车业务逻辑

代码位置：

- `backend/src/main/java/com/mall/service/MallService.java`
- 方法：`addToCart(UserAccount user, CartItemRequest request)`

执行步骤：

1. 查询商品是否存在。
2. 校验商品是否上架。
3. 解析 SKU：如果前端没有传 `skuId`，取商品下第一个可用 SKU。
4. 校验 SKU 是否启用。
5. 校验库存是否足够。
6. 查询当前用户购物车中是否已有同商品同 SKU。
7. 如果已有则累加数量。
8. 如果没有则创建新的 `CartItem`。
9. 再次校验累加后的数量不能超过库存。
10. 保存购物车项。

对应实体：

- `backend/src/main/java/com/mall/entity/CartItem.java`
- `backend/src/main/java/com/mall/entity/Product.java`
- `backend/src/main/java/com/mall/entity/ProductSku.java`

## 6. 订单生成流程

订单是在购物车结算时生成的。

### 6.1 前端购物车页面

代码位置：

- `frontend/src/views/CartView.vue`

页面加载时调用：

```js
onMounted(loadCart)
```

`loadCart` 同时拉取：

- 购物车：`getCart()`
- 优惠券：`getCoupons()`
- 会员积分：`getMemberProfile()`
- 地址列表：`getAddresses()`

### 6.2 前端结算参数

代码位置：

- `frontend/src/views/CartView.vue`
- 方法：`submitOrder()`

按钮禁用条件：

```vue
:disabled="submitting || !cart.items?.length || !shippingAddress.trim()"
```

表示：

- 正在提交时不能重复点。
- 购物车为空不能提交。
- 收货地址为空不能提交。

提交时组装：

```js
await checkout({
  shippingAddress: shippingAddress.value,
  couponId: couponId.value,
  pointsUsed: Math.max(0, Number(pointsUsed.value || 0)),
  idempotencyKey: `checkout-${Date.now()}-${Math.random().toString(16).slice(2)}`
})
```

接口封装：

- `frontend/src/api/mall.js`

```js
export const checkout = (payload) => http.post('/orders/checkout', payload)
```

最终请求：

```http
POST /api/orders/checkout
Authorization: Bearer <token>
Content-Type: application/json

{
  "shippingAddress": "张三 13800000000 北京市海淀区示例路 1 号",
  "couponId": null,
  "pointsUsed": 0,
  "idempotencyKey": "checkout-..."
}
```

### 6.3 后端结算入口

代码位置：

- `backend/src/main/java/com/mall/controller/MallController.java`

```java
@PostMapping("/orders/checkout")
public ApiResponse<OrderResponse> checkout(@AuthenticationPrincipal UserAccount user,
                                           @Valid @RequestBody CheckoutRequest request)
```

参数来源：

- 当前用户来自 JWT。
- 结算参数来自 JSON 请求体。

### 6.4 结算参数校验

代码位置：

- `backend/src/main/java/com/mall/dto/CheckoutRequest.java`

```java
@NotBlank @Size(max = 255) String shippingAddress
Long couponId
Integer pointsUsed
Long skuId
String idempotencyKey
```

含义：

- 收货地址不能为空，最大 255 个字符。
- 优惠券 ID 可选。
- 积分可选，后端会把空值当成 0，并且不允许超过用户已有积分。
- 幂等键可选，用于防止重复下单。

### 6.5 后端订单生成核心逻辑

代码位置：

- `backend/src/main/java/com/mall/service/MallService.java`
- 方法：`checkout(UserAccount user, CheckoutRequest request)`

该方法使用 `@Transactional`，确保订单生成、库存扣减、积分扣减、优惠券使用、购物车清空要么全部成功，要么全部回滚。

完整步骤如下。

#### 步骤 1：幂等校验

如果前端传入 `idempotencyKey`，后端先检查是否已处理过：

- `idempotentRecordMapper.existsByIdemKey(...)`

如果已存在，则尝试返回之前生成的订单，避免网络抖动或用户重复点击导致重复下单。

对应实体：

- `backend/src/main/java/com/mall/entity/IdempotentRecord.java`

#### 步骤 2：获取购物车

后端读取当前用户购物车：

```java
List<CartItem> cartItems = cartItemMapper.findByUserId(user.getId());
```

如果购物车为空：

```java
throw new BusinessException("Cart is empty");
```

#### 步骤 3：初始化订单主表

创建 `OrderEntity`：

- 订单号：`MALL` + UUID 片段
- 用户：当前登录用户
- 收货地址：前端传入
- 状态：`PENDING_PAYMENT`
- 创建时间：当前时间
- 幂等键：前端传入

对应实体：

- `backend/src/main/java/com/mall/entity/OrderEntity.java`

#### 步骤 4：遍历购物车，扣库存并生成订单明细

对每个 `CartItem`：

1. 获取商品和 SKU。
2. 使用 mapper 的库存扣减方法扣减 SKU 库存。
3. 同步扣减商品总库存。
4. 如果库存不足，抛出异常并回滚事务。
5. 创建 `OrderItem`。
6. 保存下单时的单价快照。
7. 累计订单原始总价。

对应实体：

- `backend/src/main/java/com/mall/entity/OrderItem.java`
- `backend/src/main/java/com/mall/entity/Product.java`
- `backend/src/main/java/com/mall/entity/ProductSku.java`

特别注意：

`OrderItem.price` 保存的是下单时的价格快照，因此后续商品改价不会影响历史订单。

#### 步骤 5：计算优惠券和积分

优惠券计算位置：

- `MallService#couponDiscount(...)`

逻辑：

1. 未传 `couponId` 则优惠为 0。
2. 校验用户是否已领取该优惠券。
3. 校验优惠券是否已经使用。
4. 校验订单金额是否达到门槛。
5. 符合条件则标记优惠券已使用，返回优惠金额。

积分计算逻辑：

1. `pointsUsed` 为空时按 0 处理。
2. 不允许使用超过用户已有积分。
3. 100 积分抵扣 1 元。
4. 积分抵扣不能超过优惠券抵扣后的剩余金额。
5. 使用积分后写入积分流水。

积分流水实体：

- `backend/src/main/java/com/mall/entity/PointRecord.java`

#### 步骤 6：保存订单、写入幂等记录、清空购物车

后端设置订单金额字段：

- `originalAmount`
- `discountAmount`
- `pointsUsed`
- `pointsDiscountAmount`
- `totalAmount`

然后保存订单：

```java
OrderEntity saved = orderMapper.save(order);
```

如果有幂等键，则保存幂等记录：

```java
idempotentRecordMapper.save(record);
```

最后清空当前用户购物车：

```java
cartItemMapper.deleteByUserId(user.getId());
```

返回：

```java
return toOrderResponse(saved);
```

响应数据由 `OrderResponse` 承载：

- `backend/src/main/java/com/mall/vo/OrderResponse.java`

## 7. 支付与订单状态流转

订单生成后状态为：

```text
PENDING_PAYMENT
```

普通用户可执行：

- 支付：`PATCH /api/orders/{orderId}/pay`
- 取消：`PATCH /api/orders/{orderId}/cancel`
- 确认收货：`PATCH /api/orders/{orderId}/receive`

代码位置：

- `backend/src/main/java/com/mall/controller/MallController.java`
- `MallController#pay`
- `MallController#cancel`
- `MallController#receive`

业务逻辑位置：

- `backend/src/main/java/com/mall/service/MallService.java`
- `pay(...)`
- `cancel(...)`
- `confirmReceived(...)`

状态约束：

- 只有 `PENDING_PAYMENT` 可以支付。
- 只有 `PENDING_PAYMENT` 可以取消，取消时恢复商品和 SKU 库存。
- 只有 `SHIPPED` 可以确认收货。

商家/管理员可执行发货：

- `PATCH /api/admin/orders/{orderId}/ship`

代码位置：

- `backend/src/main/java/com/mall/controller/AdminController.java`
- `AdminController#ship`
- `backend/src/main/java/com/mall/service/AdminService.java`
- `AdminService#shipOrder`

发货约束：

- 只有 `PAID` 状态订单可以发货。

## 8. 参数校验分层总结

项目的参数校验分为三层。

### 8.1 前端基础校验

位置：

- `LoginView.vue`
- `RegisterView.vue`
- `ProductView.vue`
- `CartView.vue`
- `MerchantView.vue`

作用：

- 提前拦截明显错误。
- 改善用户体验。
- 减少无效请求。

示例：

- 注册时校验两次密码一致。
- 商品详情页校验购买数量不超过库存。
- 购物车结算按钮在无商品或无地址时禁用。
- 商品后台表单使用 `required`、`min`、`step` 等 HTML 校验。

### 8.2 DTO Bean Validation 校验

位置：

- `backend/src/main/java/com/mall/dto/*.java`

作用：

- 保证后端接口边界安全。
- 即使绕过前端直接调用接口，也能拦截非法参数。

常用注解：

- `@NotBlank`
- `@NotNull`
- `@Size`
- `@Email`
- `@Min`
- `@DecimalMin`

### 8.3 Service 业务校验

位置：

- `AuthService`
- `MallService`
- `AdminService`

作用：

- 校验数据库状态和业务规则。
- 例如用户名唯一、邮箱唯一、库存是否足够、商品是否上架、订单状态是否允许流转、积分是否足够、优惠券是否已领取等。

## 10. 端到端链路示意

### 10.1 登录链路

```text
LoginView.vue
  -> mall.js login(payload)
  -> http.js Axios POST /api/auth/login
  -> AuthController.login(@Valid @RequestBody LoginRequest)
  -> AuthService.login()
  -> UserMapper.findByUsername()
  -> PasswordEncoder.matches()
  -> JwtService.generateToken()
  -> ApiResponse<AuthResponse>
  -> saveAuth()
  -> router.push(...)
```

### 10.2 注册链路

```text
RegisterView.vue
  -> 前端确认密码校验
  -> mall.js register(payload)
  -> POST /api/auth/register
  -> AuthController.register(@Valid @RequestBody RegisterRequest)
  -> RegisterRequest Bean Validation
  -> AuthService.register()
  -> 用户名/邮箱唯一性校验
  -> BCrypt 加密密码
  -> UserAccount 保存
```

### 10.3 添加商品链路

```text
MerchantView.vue 商品表单
  -> admin.js createProduct(productForm)
  -> http.js 添加 Authorization 和 X-Merchant-Id
  -> POST /api/admin/products
  -> SecurityConfig 校验 ADMIN/MERCHANT 权限
  -> AdminController.createProduct(@Valid @RequestBody ProductRequest)
  -> ProductRequest Bean Validation
  -> AdminService.createProduct()
  -> requireMerchantId()
  -> CategoryMapper.findById()
  -> Product 保存
  -> ProductSku 保存
  -> ProductDetailBlock 保存
  -> ProductResponse 返回
```

### 10.4 购买和订单生成链路

```text
ProductView.vue
  -> 选择 SKU + 输入数量
  -> addCart({ productId, skuId, quantity })
  -> POST /api/cart
  -> MallController.addToCart(@AuthenticationPrincipal user, @Valid @RequestBody CartItemRequest)
  -> MallService.addToCart()
  -> 商品/SKU/库存校验
  -> CartItem 保存或数量累加

CartView.vue
  -> loadCart()
  -> 填写收货地址/选择优惠券/积分
  -> checkout({ shippingAddress, couponId, pointsUsed, idempotencyKey })
  -> POST /api/orders/checkout
  -> MallController.checkout()
  -> CheckoutRequest Bean Validation
  -> MallService.checkout() @Transactional
  -> 幂等校验
  -> 读取购物车
  -> 创建 OrderEntity
  -> 遍历 CartItem 创建 OrderItem
  -> 扣减 ProductSku 和 Product 库存
  -> 计算优惠券和积分
  -> 保存订单
  -> 保存幂等记录
  -> 清空购物车
  -> 返回 OrderResponse
  -> router.push('/orders')
```
